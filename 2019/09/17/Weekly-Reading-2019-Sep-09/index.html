Weekly Reading (2019/Sep 09 ~ 15)

Sep 17, 2019

  • 別跟我稱兄道弟，我們不是一家人 : 瘋狂的 Netflix

    不近人情的要求，相對也可得到豐富的報酬，各取所需罷了。

  • Is High Quality Software Worth the Cost?

    作者 Martin Fowler) 首先將 Quality 分作 External 使用者能直接體會到的特性 (ex. 好的UI , 較少Bug) 和 Internal 使用者較無法體會的特性 (ex. 設計良好的 Software Architecture)。

    好的 UI 設計能吸引使用者，賣得好價錢。那使用者看不到的架構設計值得追求嗎？ 答案當然是 Yes!!
    優良的設計能在同樣的品質下，增減功能。 若在不良的設計下增減功能，常常會是個悲劇。

  • 谁在“谋杀” Hadoop？
    因為工作，在 Hadoop 上琢磨不少時間。 文中的一段話

    看了心有戚戚焉，在使用 Hadoop 的確踩了不少坑。
    現在關於 Big Data 的 solution，Hadoop 已經不是唯一的選擇了。

  • Cloudflare outage caused by bad software deploy
    看似簡單的 Regular expression，也是能讓系統崩潰的…

  • 一起來了解 Web Authentication
    加入了實體裝置來作為驗證途徑，在現在人手一機的世界或許會是之後的趨勢，值得花時間深入研究。

  • CockroachDB: Architecture of a Geo-distributed SQL Database
    架構上和 HBase 相似，同樣有 Leadership、Replication等概念，但主要以 Raft) 作為 consensus algorithm)。
    不同的是有所謂 Distributed Transaction 的功能。
    但從其流程來看，若寫入頻繁的話，效能會是個議題。
